# 非機能要件書（Non-Functional Requirements）

| 項目 | 内容 |
|------|------|
| **文書番号** | REQ-NFR-001 |
| **バージョン** | 1.0.0 |
| **作成日** | 2026-03-25 |

---

## 1. パフォーマンス要件

| 指標 | 目標値 | 測定条件 |
|------|--------|---------|
| API レスポンスタイム（p50） | 100ms 以内 | 通常負荷時 |
| API レスポンスタイム（p95） | 500ms 以内 | 通常負荷時 |
| API レスポンスタイム（p99） | 1000ms 以内 | ピーク負荷時 |
| スループット | 100 req/sec | 同時接続50 |
| DB クエリタイム（p95） | 50ms 以内 | インデックス使用時 |

---

## 2. 可用性要件

| 指標 | 目標値 |
|------|--------|
| システム可用性 | 99.9%（月間43分以内のダウンタイム） |
| 計画メンテナンス | 月1回、深夜2〜4時 |
| RTO（目標復旧時間） | 1時間以内 |
| RPO（目標復旧時点） | 1時間以内（日次バックアップ + WAL） |
| フェイルオーバー | 自動（DB レプリカ + Redis Sentinel） |

---

## 3. スケーラビリティ要件

| 指標 | 現在 | 将来（3年後） |
|------|------|------------|
| 同時アクティブユーザー | 50 | 200 |
| 総ユーザー数 | 600 | 1,000 |
| 日次 API コール数 | 10,000 | 50,000 |
| 監査ログ保存量 | 1GB/月 | 5GB/月 |

---

## 4. セキュリティ要件

| 項目 | 要件 |
|------|------|
| 認証 | JWT (HS256 以上)、アクセストークン有効期限 30分 |
| 認可 | RBAC、最小権限の原則 |
| 通信暗号化 | TLS 1.2 以上必須 |
| パスワード | bcrypt (cost factor 12以上) |
| レート制限 | ログイン: 5回/分、API: 100回/分 |
| CSRF 保護 | SameSite Cookie + Origin ヘッダー検証 |
| セキュリティヘッダー | CSP, HSTS, X-Frame-Options, X-Content-Type-Options |
| 脆弱性管理 | OWASP Top 10 対策、四半期ペネトレーションテスト |

---

## 5. 保守性要件

| 項目 | 要件 |
|------|------|
| テストカバレッジ | 90% 以上 |
| コードドキュメント | 全 public API に docstring 必須 |
| 依存関係更新 | 月次セキュリティパッチ適用 |
| ログレベル | ERROR/WARNING/INFO/DEBUG（本番: INFO以上） |
| モニタリング | Prometheus メトリクス + Grafana ダッシュボード |

---

## 6. 運用性要件

| 項目 | 要件 |
|------|------|
| デプロイ方法 | Docker Compose（開発）/ Kubernetes（本番） |
| ロールバック | ブルーグリーンデプロイ、5分以内にロールバック可能 |
| バックアップ | PostgreSQL 日次フルバックアップ + WAL アーカイブ |
| ログ保存期間 | アプリケーションログ: 90日、監査ログ: 7年 |
| CI/CD | GitHub Actions、全テスト通過後のみ merge 可能 |

---

## 7. 移植性・互換性要件
